# Знания: ограничения

Сюда переносите ограничения, которые важно помнить между сессиями.

## Spec-governance ограничения

- Beginner path не должен требовать тяжёлый профессиональный audit process для первой feature.
- Любое отклонение от user-spec должно быть явно записано как `DEV-*` и утверждено до статуса `approved`.
- Задача без понятного verification path считается неполной, даже если она маленькая.
- Feature closeout не считается полным, пока decisions не просмотрены и `project-knowledge-update-proposal.md` не говорит `required` или `not_required` с причиной.
