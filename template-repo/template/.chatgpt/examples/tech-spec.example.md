# Пример хорошо заполненной технической спецификации

## Краткое описание решения
- В шаблон фабрики добавляется набор примерных артефактов в папке `.chatgpt/examples/`.
- Пустые шаблоны ключевых файлов переписываются так, чтобы содержать русские подсказки по заполнению.
- В bootstrap добавляется отдельный документ, объясняющий, как соотносить пустой шаблон и пример.

## Какие компоненты затрагиваются
- `bootstrap/12-примеры-хорошего-заполнения.md`
- `template-repo/template/.chatgpt/user-spec.md`
- `template-repo/template/.chatgpt/tech-spec.md`
- `template-repo/template/.chatgpt/reality-check.md`
- `template-repo/template/.chatgpt/evidence-register.md`
- `template-repo/template/.chatgpt/task-index.yaml`
- `template-repo/template/.chatgpt/examples/*`

## Какие изменения вносятся
1. Добавляется новая папка `.chatgpt/examples/` с примерами хорошо заполненных артефактов.
2. Пустые шаблоны получают подробные русские подсказки по заполнению.
3. README и bootstrap обновляются так, чтобы пользователь видел, где брать пример и как не путать пример с реальными данными.

## Какие зависимости и ограничения учтены
- Скрипты продолжают использовать английские ключи в YAML, поэтому названия ключей не переводятся.
- Все поясняющие комментарии и примеры заполняются по-русски.
- Примерные файлы не должны мешать работе валидаторов и launcher.

## Как будет проверяться результат
- Проверить, что новые файлы примеров существуют в `.chatgpt/examples/`.
- Проверить, что пустые шаблоны содержат русские подсказки по заполнению.
- Проверить, что архив собирается без ошибок и не содержит временных тестовых каталогов.

## Что может пойти не так
- Пользователь может принять примерный файл за финальный артефакт и не заменить данные.
- Слишком подробный пример может соблазнить копировать текст без адаптации.
- Если в шаблоне оставить английский термин без пояснения, это снова создаст неоднозначность.
