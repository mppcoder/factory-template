# T-001

## Название
Уточнить whitelist временных ошибок

## Цель
Подтвердить, какие ошибки действительно должны приводить к retry.

## Основание
Раздел "Подход" в `.chatgpt/tech-spec.md`.

## Входной контекст
- `.chatgpt/reality-check.md`
- `docs/integrations.md`

## Действия
1. Сопоставить ответы API и внутренние error codes.
2. Зафиксировать whitelist.
3. Обновить документацию интеграции.

## Критерии приемки
- есть итоговый whitelist;
- список отражён в документации и reality check.

## Риски
Неверная классификация вызовет лишние retry или потерю уведомлений.

## Зависимости
Нет.
