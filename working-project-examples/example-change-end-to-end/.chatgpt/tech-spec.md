# Техническая спецификация

## Подход
1. Выделить список временных ошибок HTTP-интеграции.
2. При таких ошибках возвращать уведомление в очередь с увеличением `retry_count`.
3. Ограничить число повторов до 3.
4. Передавать один и тот же идемпотентный ключ между попытками.
5. Логировать каждую попытку как отдельную запись с общим `request_id`.

## Что меняется в коде
- модуль классификации ошибок HTTP;
- модуль постановки в очередь;
- логика отправки HTTP-запросов;
- тесты на retry-поведение.

## Что не делаем в этом change
- не добавляем UI для настройки retry;
- не меняем политику retry для других каналов;
- не вводим распределённый backoff.
