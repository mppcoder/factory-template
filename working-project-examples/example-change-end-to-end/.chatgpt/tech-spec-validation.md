# Проверка технической спецификации

## Проверка реализуемости
Подход реализуем на текущей архитектуре, потому что очередь, журнал и счётчик попыток уже существуют.

## Проверка рисков
Основной риск — неверная классификация ошибок. Он снижается whitelist-подходом и тестами.

## Проверка объёма
Объём изменения умеренный и подходит под класс `feature` в режиме `hybrid`.
