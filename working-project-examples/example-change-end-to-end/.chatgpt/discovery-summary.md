# Краткий итог discovery

## Что изучено
- текущая логика отправки уведомлений;
- журнал попыток доставки;
- обработка ответов внешнего API;
- обсуждения команды о потере уведомлений.

## Главный вывод
Нужен ограниченный retry только для временных ошибок, с фиксированным числом повторов, журналированием и защитой от дублей.
