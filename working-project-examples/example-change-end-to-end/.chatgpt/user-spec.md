# Пользовательская спецификация

## Что хочет бизнес
Нужно, чтобы уведомления не терялись из-за кратковременных отказов внешнего API.

## Что должно измениться
Система должна автоматически повторять отправку уведомления при временной ошибке, но делать это ограниченно и прозрачно.

## Что важно не сломать
- нельзя генерировать бесконтрольные дубли;
- нельзя потерять журнал попыток;
- нельзя ретраить постоянные ошибки.

## Критерий пользы
Количество потерянных уведомлений из-за кратковременных сбоев должно заметно снизиться.
