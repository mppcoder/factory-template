# Пользовательская спецификация

## Что нужно получить
Понять фактическое устройство существующей системы и зафиксировать расхождения между документацией и реальностью.

## Ожидаемый результат
- инвентаризация модулей и процессов;
- карта архитектуры как есть;
- список расхождений и рисков;
- рекомендации для следующего change.
