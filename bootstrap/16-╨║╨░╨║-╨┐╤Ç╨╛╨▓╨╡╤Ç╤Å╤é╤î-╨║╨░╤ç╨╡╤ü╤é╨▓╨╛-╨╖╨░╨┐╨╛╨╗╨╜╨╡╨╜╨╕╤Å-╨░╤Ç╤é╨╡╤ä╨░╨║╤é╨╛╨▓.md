# Как проверять качество заполнения артефактов

Этот документ объясняет, как отличать "файл существует" от "файл действительно полезен для работы".

## Главная идея

Недостаточно просто создать файл. Важно, чтобы в файле были:
- факты, а не заглушки;
- конкретика, а не общие слова;
- проверяемые критерии, а не размытые обещания.

## Что проверяется в v2.3.0

### 1. Проверка реальности (Reality Check)
Файл `.chatgpt/reality-check.md` должен содержать:
- заполненный блок о текущем состоянии проекта;
- заполненный блок по официальной документации;
- заполненный блок по реальным кейсам и known bugs;
- заполненный блок по расхождениям;
- предварительный вывод.

### 2. Реестр доказательств (Evidence Register)
Файл `.chatgpt/evidence-register.md` должен содержать:
- минимум два подтверждения;
- минимум один тег источника;
- отсутствие полностью пустых пунктов.

### 3. Реестр задач (Task Index)
Файл `.chatgpt/task-index.yaml` должен содержать:
- непустой `change.id`;
- непустой `change.title`;
- непустой `change.summary`;
- хотя бы одну задачу;
- у каждой задачи должны быть `title`, `outputs`, `acceptance`.

### 4. Validation / Verification / Done
Файлы проверки должны содержать:
- явный вывод;
- перечень замечаний или подтверждений;
- не менее одного содержательного пункта.

## Как пользоваться

1. Заполните артефакты.
2. Запустите:

```bash
./scripts/validate-quality.sh
```

3. Если проверка упала, откройте подсказки в выводе и исправьте именно слабые места, а не только наличие файла.
