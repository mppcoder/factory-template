# Анализ бага

## Обязательный порядок шагов
1. Воспроизвести проблему.
2. Собрать evidence.
3. Создать bug report.
4. Классифицировать слой дефекта.
5. При необходимости подготовить ChatGPT handoff bug note.
6. Если defect reusable — подготовить factory feedback.
7. Создать Codex fix task.
8. Провести verification.
9. Если defect reusable — подготовить sync back в фабрику.

## Канонический маршрут defect-flow
1. reproduce
2. collect evidence
3. create bug report
4. classify defect layer
5. if reusable create factory feedback
6. if needed create ChatGPT bug handoff
7. create Codex task
8. verify
9. sync reusable change back
