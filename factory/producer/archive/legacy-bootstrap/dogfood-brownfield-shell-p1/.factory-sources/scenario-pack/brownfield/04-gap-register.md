# Реестр разрывов

Зафиксируйте, где есть расхождения между:
- кодом и документацией;
- ожиданием и фактом;
- текущим и целевым состоянием.

## Правило фиксации gap / defect
Любой gap, найденный в reverse engineering или brownfield-анализе, должен стать bug report или structured defect report до remediation planning.

## Правило brownfield-gap
Любой существенный gap, найденный в reverse engineering, должен стать bug report или structured defect report до remediation planning.
