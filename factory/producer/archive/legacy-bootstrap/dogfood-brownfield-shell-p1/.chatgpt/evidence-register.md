# Реестр доказательств

> Подсказка: сюда вносятся основания, на которых строится решение.
> Каждый пункт должен быть коротким, конкретным и подтверждаемым.
> Минимум: два содержательных пункта с разными тегами источников.

## Допустимые теги
- PROJECT — подтверждено внутри текущего проекта
- FIX — подтверждено ранее зафиксированным решением проекта
- DOC — подтверждено официальной документацией
- REAL — подтверждено реальным кейсом, issue или обсуждением
- ASSUMPTION — пока остается предположением

## Как заполнять
- Начинайте строку с тега в квадратных скобках, например: `[PROJECT]`.
- После тега пишите конкретный факт, ссылку на файл, конфиг, issue или решение.
- Не оставляйте пустых тегов и не копируйте примеры без адаптации.
