# Сквозной пример change lifecycle

## Где смотреть полный пример
См. папку:
`working-project-examples/example-change-end-to-end/`

## Что в ней есть
- intake и project profile;
- policy status и stage state;
- reuse, reality, conflict и evidence;
- user spec и tech spec;
- task index;
- готовый work item с задачами;
- verification и done.

## Когда использовать
Открывайте этот пример, если нужно быстро показать команде полный цикл жизни одного изменения, а не отдельный шаблон.
