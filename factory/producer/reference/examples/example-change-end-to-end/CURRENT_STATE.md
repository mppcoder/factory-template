# Текущее состояние проекта

## Что уже есть
- веб-приложение с модулем отправки уведомлений;
- очередь доставки;
- журнал попыток отправки;
- интеграция с внешним HTTP API.

## Что подтверждено
- временные ошибки внешнего API сейчас не приводят к автоматическому повтору;
- часть уведомлений теряется при кратковременных сбоях.

## Что предположительно
- часть логики retry уже обсуждалась, но не была доведена до продакшена.

## Что неясно
- есть ли лимит по количеству повторов на уровне внешнего API;
- есть ли требование к идемпотентности на стороне получателя.

## Следующий обязательный шаг
Провести reuse и reality check перед формированием change spec.
