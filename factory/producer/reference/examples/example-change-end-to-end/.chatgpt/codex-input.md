# Пакет передачи в Codex

## Что нужно сделать
Реализовать ограниченный retry для временных ошибок внешнего HTTP API с сохранением идемпотентного ключа.

## На чем основано решение
- [PROJECT] В проекте уже есть журнал попыток и поле `retry_count` для email-канала.
- [DOC] Документация внешнего API разрешает retry для `429`, `503` и сетевых сбоев.
- [REAL] Реальные инциденты подтверждают потерю уведомлений без retry.

## Какие артефакты считать источником правды
- `.chatgpt/reality-check.md`
- `.chatgpt/tech-spec.md`
- `.chatgpt/task-index.yaml`
- `docs/integrations.md`

## Что проверить после реализации
- ограничение количества повторов;
- сохранение идемпотентного ключа;
- журналирование всех попыток;
- прохождение тестов по retry-пути.
