# Отчёт о конфликтах

## Найденные конфликты
1. [PROJECT] Текущая логика считает любое неуспешное HTTP-ответом финальной ошибкой.
2. [DOC] Документация API допускает повтор только для части кодов и сетевых ошибок.
3. [ASSUMPTION] Без идемпотентного ключа ретраи могут породить дубли.

## Как разрешён конфликт
- вводим whitelist временных ошибок;
- сохраняем идемпотентный ключ между попытками;
- ограничиваем число повторов до трёх;
- все попытки журналируем.
