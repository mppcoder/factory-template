# Пример хорошо заполненной спецификации потребности пользователя

## Краткое название изменения
- «Добавить в фабрику проектов примеры хорошо заполненных артефактов для ключевых этапов»

## Что нужно сделать
- Нужно дополнить шаблон фабрики не только пустыми файлами, но и примерами качественного заполнения для `user-spec`, `tech-spec`, `reality-check`, `evidence-register` и `task-index`.
- Примеры должны помогать быстрее стартовать новый проект и уменьшать число ошибок при первом заполнении.
- Все пояснения внутри файлов должны быть на русском языке.

## Для кого это делается
- Для владельца проекта, который запускает фабрику впервые.
- Для инженера, который готовит материалы к handoff в Codex.
- Для аналитика, который оформляет discovery и хочет опираться на образец хорошего результата.

## Как понять, что результат полезен
- В шаблоне появились отдельные примерные файлы.
- По ним можно понять не только структуру, но и ожидаемую глубину заполнения.
- Внутри шаблонов нет необъяснённых английских формулировок.

## Что точно входит в объём
- Обновление пустых шаблонов подробными русскими подсказками.
- Добавление папки `.chatgpt/examples/` с примерными файлами.
- Добавление bootstrap-документа, объясняющего, как пользоваться примерами.

## Что точно не входит в объём
- Не входит автоматическая генерация содержимого через LLM.
- Не входит доменная адаптация только под domain reference.
- Не входит изменение core-pipeline фабрики.

## Риски и ограничения
- Пример не должен подменять собой реальное содержание проекта.
- Нельзя оставлять в боевом проекте примерные данные без замены на реальные.
