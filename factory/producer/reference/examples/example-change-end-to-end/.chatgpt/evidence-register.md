# Реестр доказательств

- [PROJECT] В проекте уже есть поле `retry_count` и журнал попыток доставки.
- [FIX] Ранее аналогичный паттерн ограниченного retry использовался для email-канала.
- [DOC] Документация внешнего API разрешает retry для `429`, `503` и сетевых сбоев при использовании идемпотентного ключа.
- [REAL] В реальных инцидентах команда наблюдала потерю уведомлений при кратковременных отказах API.
- [ASSUMPTION] Порог в три повтора достаточен для снижения потерь без избыточных дублей.
