# Release 2.5 Success Metrics

## Scope

Эта таблица задаёт единые KPI для двух треков `2.5-A` и `2.5-B`.
Метрики обязательны для gate-проверок в `docs/releases/2.5-roadmap.md`.
Release truth source: `docs/releases/release-scorecard.yaml`.
Current scorecard result: RC evidence is green; GA KPI confirmation is still pending.

## Metric Table

| Metric ID | Metric | Why It Matters | MVP Threshold | Full 2.5 Threshold | Primary Evidence |
|---|---|---|---|---|---|
| `M25-01` | Time-to-first-success (beginner) | Показывает реальный порог входа | <= `45` минут | <= `25` минут | scripted onboarding run + verify logs |
| `M25-02` | First-run completion rate (beginner path) | Проверяет UI-friendly и ясность шагов | >= `80%` | >= `92%` | controlled pilot checklist |
| `M25-03` | Manual intervention count per first run | Чем меньше, тем проще путь для новичка | <= `3` | <= `1` | run notes + support prompts register |
| `M25-04` | Downstream safe-sync success rate | Защищает battle repos от regressions | >= `95%` | >= `99%` | downstream sync verification reports |
| `M25-05` | Critical defects in 2.5 scope | Контроль инженерного качества | `0` открытых критических | `0` открытых критических | `reports/bugs/*.md` + verification summary |
| `M25-06` | Handoff rework loops (per task) | Показатель качества framing и routing | <= `2` | <= `1` | `.chatgpt` handoff history |
| `M25-07` | Release-facing doc consistency | Защищает от расхождения docs vs execution | 100% обязательных документов синхронизированы | 100% + без follow-up drift | checklist + review pass |
| `M25-08` | Repo-first routing compliance | Гарантирует соблюдение master-router first step | 100% | 100% | task evidence + review logs |

## Measurement Notes

- Метрики `M25-01`..`M25-03` относятся в основном к треку `2.5-B`.
- Метрики `M25-04`..`M25-06` относятся в основном к треку `2.5-A`.
- Метрики `M25-07` и `M25-08` являются сквозными и обязательны для обоих треков.
- Если хотя бы одна метрика ниже порога, соответствующий gate `G25-*` считается не закрытым.

## Gate Mapping

1. `G25-RC` требует зеленой RC evidence baseline: consolidated verify, novice onboarding smoke, downstream upgrade/rollback UX.
2. `G25-GA` требует все `M25-*` на full-порогах и явный release decision.
3. Если `docs/releases/release-scorecard.yaml` говорит `ga_ready: false`, markdown-отчеты не должны объявлять GA-ready статус.
