# Дорожная карта полевого пилота 2.5.1

## Область

Этот roadmap описывает post-2.5.0 полевую проверку. Он не переоткрывает scorecard `2.5.0 GA Ready`; его задача - отделить repo-controlled GA evidence от proof на реальных проектах.

- Проверяемая линия: `2.5.1-field-pilot`
- Исходный релиз: `2.5.0`
- Pipeline stage: `audit -> evidence -> gap-capture`
- Текущий статус: `pending field proof`
- Статус field evidence: `no-field-evidence`
- Defect capture path: `reports/bugs/YYYY-MM-DD-field-pilot-evidence-gap.md`

## Общие pass criteria

Field pilot считается пройденным только если одновременно выполнено:

- все пять scenario files имеют статус `passed`;
- каждый сценарий фиксирует реальные project identifiers или approved anonymized equivalent для repo storage;
- каждый сценарий фиксирует команды, даты, operator notes и retained artifacts;
- каждый defect или gap из пилота имеет report в `reports/bugs/`;
- downstream sync cycle 2 доказывает, что второй update применяется после cycle 1 без потери project-owned state;
- `reports/release/2.5-field-pilot-evidence.md`, `TEST_REPORT.md`, `CURRENT_FUNCTIONAL_STATE.md` и `RELEASE_CHECKLIST.md` обновлены из `pending` в measured result.

## Общие fail criteria

Field pilot fails или остаётся blocked, если:

- реальный проект недоступен и approved anonymized evidence package тоже отсутствует;
- сценарий требует manual intervention без документации и classification;
- repo-first routing пропущен в field run;
- sync tooling перезаписывает project-owned files;
- rollback не восстанавливает ожидаемое состояние;
- два downstream sync cycles нельзя выполнить на одной downstream lineage.

## Примечания к исполнению

Не отмечать field evidence как `passed` на основе local synthetic runs. Если pilot project приватный, в repo коммитятся только sanitized evidence: тип проекта, scenario id, summary команд, measured KPI, artifact inventory и defect links.
