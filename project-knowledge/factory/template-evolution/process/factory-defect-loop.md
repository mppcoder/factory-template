# Цикл обработки фабричного дефекта

1. В working project найден defect.
2. Создан bug report.
3. Классификация указывает `factory-template` или `shared/unknown`.
4. Создан factory feedback.
5. Прогнан drift check.
6. Подготовлен Codex patch/task pack.
7. Patch проверен.
8. Выполнен safe apply или подготовлен ручной sync.
9. Обновлены registry и история изменений.
