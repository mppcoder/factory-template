# chg-20260423-021

## Заголовок
Default factory-template ChatGPT Project to no unless instruction contract changed

## Что сделано
- completion-package rules уточнены: contour `factory-template ChatGPT Project` больше не считается default impact, если canonical repo/path/entrypoint/instruction text не менялись
- source-of-truth scenarios и generator теперь прямо говорят: по умолчанию `нет`, а `да` допустимо только при реальном изменении instruction contract
- validator закрепляет новую формулировку, чтобы generic completion output не размывал repo-first contract

## Почему это важно
Теперь closeout не подталкивает к лишнему обновлению factory-template ChatGPT Project там, где короткая repo-first инструкция остается прежней и source-of-truth уже стабильно живет в репо.
