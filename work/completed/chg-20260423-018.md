## Заголовок
Make VS Code Codex handoff UI-first by default

## Что сделано
- source-of-truth routing contract переведен на dual-path: `manual-ui (default)` для интерактивного VS Code Codex extension и `launcher-first strict mode (optional)` для automation/reproducibility/shell-first
- generated `.chatgpt` handoff/completion artifacts, validators и template stubs синхронизированы с новой схемой
- исправлен incidental parser defect: structured handoff YAML теперь корректно читается даже если идет после markdown heading, поэтому explicit routing overrides не теряются
- оформлены и закрыты reusable bug reports/factory feedback для bug-018 и bug-019

## Почему это важно
Теперь шаблон не заставляет пользователя extension-first workflow идти в terminal без необходимости, но при этом сохраняет строгий executable launch path для воспроизводимых запусков и не обещает auto-switch внутри уже открытой live session.
